# CE-ESY-assignments
1. هيكل البيانات (The Structure)
يستخدم الكود struct يسمى CircularBuffer يحتوي على:

data_array: مصفوفة ثابتة الحجم (10 عناصر) لتخزين البيانات.

write_index: مؤشر يحدد مكان كتابة العنصر القادم.

read_index: مؤشر يحدد مكان قراءة العنصر القادم.

current_count: عداد لمتابعة عدد العناصر الموجودة حالياً لتجنب الامتلاء أو الفراغ الزائد.

2. العمليات الأساسية (Core Functions)
initBuffer: تهيئة المؤشرات والعداد إلى الصفر عند بدء التشغيل.

isFull & isEmpty: دوال برمجية للتحقق من حالة المخزن قبل الإضافة (لتجنب الـ Overflow) أو المسح (لتجنب الـ Underflow).

writeBuffer: تقوم بإضافة حرف للمصفوفة ثم تحريك مؤشر الكتابة خطوة للأمام. باستخدام عملية المتبقي (Modulo %)، يعود المؤشر إلى الصفر عند وصوله لنهاية المصفوفة، مما يخلق التأثير "الدائري".

readBuffer: تقوم باستخراج الحرف الذي عليه الدور، ثم تحريك مؤشر القراءة بنفس الطريقة الدائرية.

3. تدفق البرنامج (Program Flow)
يطلب البرنامج من المستخدم إدخال اسمه.

يتم دمج لاحقة نصية CE-ESY مع الاسم المُدخل.

يتم تمرير النص الناتج إلى المخزن المؤقت حرفاً بحرف.

ملاحظة هامة: إذا زاد طول النص عن حجم المخزن (10 أحرف)، سيقوم البرنامج بطباعة تحذير [Overflow] ولن يتم تخزين الأحرف الزائدة.

في النهاية، يقوم البرنامج بقراءة الأحرف المخزنة وطباعتها حتى يفرغ المخزن تماماً.
